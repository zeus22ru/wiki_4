# Презентация БочкарИИ (Reveal.js)

Современная HTML-презентация для коллег: что такое БочкарИИ, какие задачи закрывает, как выглядит, что под капотом.

## Как открыть

**Рекомендуемый способ — локальный HTTP-сервер из папки `docs/`:**

```powershell
cd docs
python -m http.server 8080
# открыть http://localhost:8080/presentation/
```

Это надёжнее двойного клика — современные браузеры под `file://` иногда блокируют относительные пути к `../images/` (см. *file URL CORS*).

Reveal.js, Mermaid и Google Fonts подгружаются с jsDelivr/Google CDN, значит **на момент демо нужен интернет**. Если демо без интернета — скажите, добавим оффлайн-сборку с локальными копиями библиотек в `vendor/`.

## Управление

- **→ / Space / N** — следующий слайд
- **← / P** — предыдущий
- **Esc / O** — overview-режим (плитка всех слайдов)
- **S** — заметки докладчика (открывается в отдельном окне)
- **F** — полноэкранный режим
- **?** — справка по горячим клавишам

## Печать в PDF (для рассылки)

1. Открыть в браузере с параметром `?print-pdf`, например `index.html?print-pdf`.
2. `Ctrl + P` → «Сохранить как PDF» → ориентация **альбомная**, поля **none / minimum**, фон **включён**, масштаб **100%**.
3. Сохранить.

Подробности — [официальная документация Reveal.js](https://revealjs.com/pdf-export/).

## Скриншоты

Все шесть скриншотов лежат в [`../images/`](../images/) и переиспользуются в [`docs/user_guide.md`](../user_guide.md). Чтобы их обновить — запустите `python scripts/capture_screenshots.py` из корня репозитория (нужен Playwright и работающее приложение, см. [`scripts/capture_screenshots.py`](../../scripts/capture_screenshots.py)).

**Python 3.14 и Playwright:** на Windows для `greenlet` может не быть готового wheel — тогда зависимости ставятся в виртуальное окружение на **Python 3.12** (в репозитории уже может быть папка `.venv`):

```powershell
& "$env:USERPROFILE\AppData\Roaming\uv\python\cpython-3.12.13-windows-x86_64-none\python.exe" -m venv .venv
.\.venv\Scripts\pip install -r requirements-dev.txt
.\.venv\Scripts\python -m playwright install chromium
$env:DEMO_USER = "ваш_логин"
$env:DEMO_PASSWORD = "ваш_пароль"
.\.venv\Scripts\python scripts\capture_screenshots.py
```

## Дизайн

- Палитра — фирменная (логотип, UI приложения): `#7c5a22`, `#a58250`, `#f4f1eb`, `#25211d`.
- Шрифты: Inter (текст) + JetBrains Mono (код), оба с Google Fonts.
- Стили — в [`styles.css`](styles.css), вся компоновка слайдов внутри `.bk-*` классов в `index.html`.
- Mermaid использует `base`-тему с переопределёнными цветами под палитру.

## Что править вручную

- В Q&A-слайде поправьте контакты докладчика (`bk-qa__contacts`).
- В заголовке титульного слайда — дата, при необходимости имя докладчика.
- Если после смены скриншотов поменялись их пропорции — `aspect-ratio` в `.bk-demo__shot` (по умолчанию `1024 / 640`).
