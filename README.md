# Векторная база знаний для вопрос-ответной системы

Система для создания векторной базы данных из HTML файлов и вопрос-ответной системы на базе знаний с использованием ollama (Qwen) и ChromaDB.

## Структура проекта

```
wiki_4/
├── data/                    # Исходные данные (HTML файлы)
│   └── pages/              # Экспортированные страницы XWiki
├── create_vector_db.py     # Скрипт для создания векторной БД
├── qa_system.py            # Скрипт вопрос-ответной системы
├── requirements.txt        # Зависимости Python
├── chroma_db/              # Векторная база данных (создается автоматически)
└── README.md               # Этот файл
```

## Требования

- Python 3.8+
- Docker с запущенным ollama (модель Qwen)
- Установленные зависимости из requirements.txt

## Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Убедитесь, что ollama запущен в Docker:

```bash
# Проверка доступности ollama
curl http://localhost:11434/api/tags
```

Если ollama не запущен, запустите его:

```bash
docker run -d -p 11434:11434 --name ollama ollama/ollama
docker exec -it ollama ollama pull qwen
```

## Использование

### 1. Создание векторной базы данных

Запустите скрипт для создания векторной базы данных из HTML файлов:

```bash
python create_vector_db.py
```

Скрипт выполнит следующие действия:
- Просканирует папку `data/pages/` и найдет все HTML файлы
- Извлечет текст из HTML файлов
- Разобьет текст на чанки (по 500 символов с перекрытием 50)
- Сгенерирует эмбеддинги через ollama (Qwen)
- Сохранит векторную базу данных в папку `chroma_db/`

### 2. Вопрос-ответная система

#### Интерактивный режим

Запустите систему в интерактивном режиме:

```bash
python qa_system.py
```

Система предложит вводить вопросы и будет генерировать ответы на основе найденных документов.

Для выхода введите: `exit`, `quit`, `выход` или `q`

#### Одиночный запрос

Вы также можете задать вопрос напрямую из командной строки:

```bash
python qa_system.py "Как настроить принтер на ТСД?"
```

## Конфигурация

Конфигурация находится в начале каждого скрипта:

### create_vector_db.py

```python
OLLAMA_URL = "http://localhost:11434"  # URL ollama
OLLAMA_MODEL = "qwen"                  # Модель для эмбеддингов
CHROMA_PERSIST_DIR = "./chroma_db"     # Папка для хранения БД
DATA_DIR = "./data/pages"              # Папка с HTML файлами
CHUNK_SIZE = 500                       # Размер чанка в символах
CHUNK_OVERLAP = 50                     # Перекрытие чанков
```

### qa_system.py

```python
OLLAMA_URL = "http://localhost:11434"  # URL ollama
OLLAMA_MODEL = "qwen"                  # Модель для генерации ответов
CHROMA_PERSIST_DIR = "./chroma_db"     # Папка с векторной БД
TOP_K_RESULTS = 3                      # Количество релевантных документов
```

## Архитектура

1. **Извлечение текста**: BeautifulSoup парсит HTML файлы и извлекает чистый текст
2. **Чанкование**: Текст разбивается на фрагменты для лучшего поиска
3. **Эмбеддинги**: Ollama генерирует векторные представления текста
4. **Векторный поиск**: ChromaDB выполняет семантический поиск по запросу
5. **Генерация ответа**: Ollama формирует ответ на основе найденных документов

## Устранение неполадок

### Ollama недоступен

Убедитесь, что Docker контейнер с ollama запущен:

```bash
docker ps | grep ollama
```

Если не запущен:

```bash
docker start ollama
```

### Ошибка при генерации эмбеддингов

Проверьте, что модель Qwen загружена:

```bash
docker exec -it ollama ollama list
```

Если модели нет, загрузите её:

```bash
docker exec -it ollama ollama pull qwen
```

### База данных не найдена

Убедитесь, что вы сначала запустили `create_vector_db.py` для создания базы данных.
